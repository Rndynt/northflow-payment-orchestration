# Release Readiness: v0.4.0 — Contract Freeze

**Phase:** S10.4 — OpenAPI / SDK contract freeze  
**Date:** 2026-06-09  
**Status:** ✅ Contract frozen — ready for external integration

---

## Summary

S10.4 completes the API contract freeze for `northflow-payment-orchestration` v0.4.0. This phase:

1. Audited every service route against the OpenAPI spec and SDK — 20 missing routes documented.
2. Fixed 4 SDK response-shape mismatches (extra `{ data }` wrapping on list/sync/upsert/options).
3. Preserved `created` on upsert by embedding it inside the `data` envelope.
4. Removed `deleteProviderAccountMethod` from SDK (no backing route or use case).
5. Added 6 merchant webhook management methods to the SDK.
6. Added comprehensive OpenAPI parity and SDK response-shape contract tests.
7. Produced `route-scope-matrix.md` and `error-contract.md` as integration references.

### ✅ Breaking Changes

None. S10.4 is a contract freeze (documentation + test) release. The only behavioral change
is the upsert route now embeds `created` inside the `data` object instead of alongside it at
the top level. This is a non-breaking fix — callers using the SDK transparently benefit from
the corrected type. Callers parsing raw HTTP responses should update to read `data.created`
instead of the top-level `created` field.
